  • Config via env vars — no YAML authoring needed

Command-Line Interface

krknctl

A developer-friendly CLI that wraps krkn-hub scenarios — providing auto-completion, validation, and structured output without touching containers directly.

  • Shell auto-completion out of the box
  • Input validation before scenario runs
  • Structured JSON/YAML output for CI pipelines
  • Graphs scenario dependencies and execution order
terminal
Kubernetes Native

Krkn Operator

A Kubernetes Operator that orchestrates chaos via Custom Resource Definitions — cloud-native, declarative, and designed for multi-cluster environments.

  • Declarative chaos via CRDs — no CLI needed
  • Target cluster credentials never shared with users
  • Control plane and target cluster are separate
  • Native Kubernetes RBAC for access control

Intelligent Automation

Krkn AI

Uses evolutionary algorithms guided by your SLOs and health checks to automatically discover and rank the most impactful chaos experiments for your cluster.

  • Generates experiments from cluster topology
  • Guided by real SLOs — not random failures
  • Ranks scenarios by resilience impact score
  • Surfaces previously unknown failure modes

Visual Control Center

Krkn Dashboard

A browser-based UI sitting on top of krkn-hub — run scenarios, watch live output, and replay or compare past experiments without touching the CLI.

  • Pick scenarios from a UI — no command memorization
  • Live output streaming as runs execute
  • Save and replay past experiments
  • Elasticsearch + Grafana integration for history
AI-Powered Assistant

Krkn Assist

A conversational CLI assistant that wraps krknctl — guiding you through scenario selection, configuration and execution through natural language interaction.

  • Describe your goal — it picks the right scenario
  • Walks you through config interactively
  • No manual YAML editing or flag memorization
  • Lowers the barrier for teams new to chaos engineering
